## Bugs Found

### LOW: `auto_save.py` -- AttributeError when Skill tool input is explicitly None

**Severity:** LOW
**File:** `ron_skills/loreconvo/hooks/scripts/auto_save.py:74`
**Commit introduced:** 7dbd4ce

**Description:** The code does:
```python
skill_name = block.get("input", {}).get("skill")
```
If a `tool_use` block has `input` set to `None` (not missing, but explicitly null), `dict.get("input", {})` returns `None` because the key exists. Then calling `.get("skill")` on `None` raises `AttributeError`.

**Impact:** In practice, Claude Code always provides a dict for `input`, so this does not trigger in normal operation. It would only affect malformed or manually-constructed transcripts. However, it is a defensive coding gap.

**Suggested fix for Ron:**
```python
input_val = block.get("input") or {}
skill_name = input_val.get("skill") if isinstance(input_val, dict) else None
```

**Test verifying this:** `tests/test_auto_save.py::TestParseTranscript::test_skill_with_no_skill_key_falls_back_to_Skill` (currently passes because the test uses `{}` not `None`)
