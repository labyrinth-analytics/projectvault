## Findings

### MEG-031: MEDIUM - SQL Optimizer version mismatch in docs

**Location**: ron_skills/sql_query_optimizer/

CLAUDE.md says version `v0.1.0` but SKILL.md says `1.0.0`. These should be synchronized. Recommend aligning to `0.1.0` since the product is still IN PROGRESS.

### MEG-032: LOW - FTS schema drift between database.py and auto_save.py

**Location**: ron_skills/loreconvo/

The FTS5 virtual table schema differs between the two files:
- `database.py` (line 61-63): `fts5(title, summary, decisions, content=sessions, content_rowid=rowid)`
- `auto_save.py` (line 174-176): `fts5(title, summary, decisions, artifacts, open_questions, content='sessions', content_rowid='rowid')`

auto_save.py includes extra columns (`artifacts`, `open_questions`) and uses string-quoted parameters. If both files create the table in different sessions, the first definition wins and the second is silently ignored. This hasn't caused bugs yet because database.py typically runs first, but the schemas should be unified to prevent future confusion.

### MEG-033: LOW - auto_save.py ensure_tables schema does not include NOT NULL on id

**Location**: ron_skills/loreconvo/hooks/scripts/auto_save.py line 153

The `ensure_tables()` function in auto_save.py still uses the old schema `id TEXT PRIMARY KEY` without `NOT NULL`. While SQLite treats PRIMARY KEY as implicitly NOT NULL in most cases (and auto_save.py always provides a session_id), this should be synchronized with database.py's new `id TEXT PRIMARY KEY NOT NULL` for consistency.
