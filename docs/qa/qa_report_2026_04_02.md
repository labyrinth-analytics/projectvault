## Findings

### MEG-036: MEDIUM - Cross-file test pollution in SQL Optimizer API tests

**Location**: ron_skills/sql_query_optimizer/api/tests/test_credits_concurrency.py

**Description**: Two tests (`test_corrupted_json_handled` and `test_missing_keys_field`) fail when run alongside `test_credits.py` but pass in isolation. Root cause: both test files manipulate `os.environ["DATA_DIR"]` at module level, but `credits.py` evaluates `DATA_DIR = Path(os.getenv("DATA_DIR", "./data"))` only once at import time. When pytest imports `test_credits.py` first, the `DATA_DIR` module constant is set to that file's temp directory. When `test_credits_concurrency.py` is subsequently imported, its `os.environ["DATA_DIR"]` override has no effect because the credits module is already cached in `sys.modules`.

**Impact**: Tests pass individually but fail in aggregate runs. This makes CI/CD unreliable if both files run together.

**Fix recommendation**: The `test_credits_concurrency.py` file should patch `credits.DATA_DIR` and `credits.CREDITS_FILE` directly (as demonstrated in the new `test_credits_isolation.py`), rather than relying on `os.environ` to propagate to already-imported module constants.

**Severity**: MEDIUM -- does not affect production code, but undermines test suite reliability.
