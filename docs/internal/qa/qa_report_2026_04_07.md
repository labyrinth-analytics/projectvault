## Code Walkthrough: fbfdd11 -- DB Discovery Fix

### Change Summary

Both `_find_loreconvo_db()` (save_to_loreconvo.py) and `_find_loredocs_db()`
(query_loredocs.py) previously built a candidates list with VM home first:

    candidates = [os.path.expanduser("~/.loreconvo/sessions.db")]
    candidates += sorted(glob.glob("/sessions/*/mnt/.loreconvo/sessions.db"))

The fix reverses this order -- mounted paths come first, VM home is appended as fallback:

    candidates = sorted(glob.glob("/sessions/*/mnt/.loreconvo/sessions.db"))
    candidates += [os.path.expanduser("~/.loreconvo/sessions.db")]

### Correctness Assessment: GREEN

The fix correctly solves the root cause. In Cowork VMs, `os.path.expanduser("~")`
expands to the ephemeral VM home (e.g. `/sessions/kind-quirky-mendel/`) rather than
Debbie's Mac home. Any writes that went to the VM home were lost at session end.
The mounted path `/sessions/*/mnt/.loreconvo/` is backed by Debbie's Mac filesystem
and persists across sessions. Swapping the order ensures agents write to the durable
path when running in a Cowork VM. The VM home fallback is retained for Claude Code on
the Mac, where `~` correctly resolves to the Mac home. The fix is symmetric across
both products. Logic is clean, no unused code, no ASCII violations. CLEAN.

### Advisory Finding: MEG-053

**Severity:** ADVISORY
**Status:** NEW
**Files:** save_to_loreconvo.py line 49, query_loredocs.py line 57

When multiple active Cowork VMs exist simultaneously, `sorted(glob.glob("/sessions/*/mnt/..."))` 
returns all matching session directories sorted lexicographically by session ID 
(e.g. `kind-quirky-mendel` before `sharp-adoring-dijkstra`). The first result is used.

This means if two Cowork sessions are active concurrently, the script picks whichever
VM's mount sorts first alphabetically -- not necessarily the current VM's mount. In a
single-user workflow with one active session at a time, this is harmless. If two
sessions are open simultaneously, a write from session B could go to session A's mount.

This is a pre-existing design limitation, not introduced by this commit. Low likelihood
in normal use. Advisory only -- no action required unless multi-session concurrent use
becomes common.

**Suggested improvement (optional, not urgent):**
Add a preference for the current session's own mount path by checking if a path
starts with `/sessions/<current-session-id>/`:

    import os, re
    # Prefer paths matching the current VM's own session dir
    # e.g. /sessions/kind-quirky-mendel/mnt/.loreconvo/
    current = os.environ.get("SESSION_ID", "")
    # Sort: current session path first, then others alphabetically
    mounts = sorted(glob.glob(...))
    own = [p for p in mounts if current and current in p]
    others = [p for p in mounts if not (current and current in p)]
    candidates = own + others + [vm_home_fallback]

However, this requires a reliable SESSION_ID env var. Given single-user single-session
normal usage, this is not urgent.
