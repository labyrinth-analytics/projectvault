## Findings

### MEG-055: ADVISORY - No timeout on claude invocation in run_agent_code.sh

- **Location:** scripts/run_agent_code.sh line 67-72
- **Description:** The `claude --print` invocation has no timeout guard. If a scheduled
  agent session hangs (network issue, model API delay, infinite loop), the script will
  run indefinitely. On a cron schedule, a hung session could block subsequent runs of
  the same agent and accumulate API charges without bound.
- **Impact:** Runaway API charges; overlapping agent sessions if cron fires again before
  prior run completes; disk growth if log file continues writing.
- **Remediation:** Wrap the invocation in a timeout (macOS `gtimeout` from coreutils,
  or prefix with `timeout` if GNU coreutils is available). Example:
  ```bash
  timeout 1800 "$CLAUDE_BIN" \
      --print \
      --permission-mode bypassPermissions \
      --output-format text \
      < "$SKILL_FILE" \
      >> "$LOG_FILE" 2>&1
  ```
  1800 seconds (30 minutes) is a reasonable upper bound for any agent session.
- **Severity:** ADVISORY
- **Test coverage:** No test -- shell script behavior

### MEG-052: YELLOW - Combined pytest run fails due to __init__.py namespace clash

- **Location:** ron_skills/loreconvo/tests/, ron_skills/loredocs/tests/, tests/
- **Description:** Running all three test suites in a single pytest invocation fails at
  collection. The __init__.py files across suites create conflicting package namespaces.
- **Impact:** Cannot do a single repo-wide pytest run. Must run each suite separately.
- **Remediation:** Remove __init__.py from test directories and use pytest rootdir/src
  layout, or add pyproject.toml [tool.pytest.ini_options] with testpaths.
- **Severity:** YELLOW (blocks CI convenience but not individual suite runs)
