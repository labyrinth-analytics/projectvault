### MEG-043 -- NEW (YELLOW) -- tests/scripts test path bug breaks two new test suites

**Severity:** YELLOW (tests cannot run, no false-negatives yet but coverage is lost)
**Files:** `tests/scripts/test_generate_license_key.py`, `tests/scripts/test_migrate_lore.py`
**Found:** 2026-04-05 (commit 9ef2954 added both files)

**Problem:**
Both files attempt to import the scripts they test from the same directory
(`Path(__file__).parent`), but the actual scripts live in `scripts/`:

- `test_generate_license_key.py` line 30: `_SCRIPT_PATH = Path(__file__).parent / "generate_license_key.py"`
  Actual location: `scripts/generate_license_key.py`
- `test_migrate_lore.py` line 10: `sys.path.insert(0, str(Path(__file__).parent))`
  Then `import migrate_lore` -- actual location: `scripts/migrate_lore.py`

**Error:**
```
FileNotFoundError: .../tests/scripts/generate_license_key.py
ModuleNotFoundError: No module named 'migrate_lore'
```

**Fix (Ron):** Update path references to point to the actual `scripts/` directory.
For `test_generate_license_key.py`:
```python
_SCRIPT_PATH = Path(__file__).resolve().parent.parent.parent / "scripts" / "generate_license_key.py"
```
For `test_migrate_lore.py`:
```python
sys.path.insert(0, str(Path(__file__).resolve().parent.parent.parent / "scripts"))
```
These are 273 and 279 line test files respectively -- the coverage is valuable
once the import paths are fixed.

### MEG-044 -- NEW (YELLOW) -- test_readme_tools.py regex misses @mcp.tool() without name=

**Severity:** YELLOW (test passes with incorrect assertion, tool count silently wrong)
**File:** `ron_skills/loredocs/tests/test_readme_tools.py`
**Found:** 2026-04-05

**Problem:**
`test_tool_count_matches_34` and `test_readme_claims_34_tools` assert 34 tools,
but LoreDocs now has 35. The test appears to pass because the detection regex
`r'@mcp\.tool\(\s*name\s*=\s*"(\w+)"'` only matches tools decorated with an
explicit `name=` argument. The newly added `get_license_tier()` uses `@mcp.tool()`
without `name=`, so it is invisible to the regex.

**Evidence:**
- `grep -c "@mcp.tool" ron_skills/loredocs/loredocs/server.py` = 35
- The test regex finds only 34 (misses `get_license_tier` at line 1445)
- `test_tool_count_matches_34` PASSES but with a wrong count

**Fix (Ron):**
1. Update the test regex to also match `@mcp.tool()` (no-args form):
   ```python
   # Count all @mcp.tool usages regardless of arguments
   names = re.findall(r'@mcp\.tool', self.server_src)
   ```
2. Update the expected count to 35 in `test_tool_count_matches_34` and
   update the string assertion in `test_readme_claims_34_tools` to `"35 MCP tools"`.
3. These fixes should go together with the doc updates in MEG-045.
